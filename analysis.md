# COVID-19 Twitter Dataset 분석 보고서 (Analysis Report)

본 문서는 Assignment 2 *covid-twitter-analysis* 프로젝트의 데이터 분석 과정을 정리한 보고서입니다.  
데이터 이해, 처리 방식, 주요 분석 결과를 중심으로 작성되었습니다.

---

1. 데이터 개요

본 프로젝트는 Kaggle에서 제공된 COVID-19 Twitter 데이터셋을 기반으로 한다.  
데이터는 2020년 3월부터 2022년 2월까지의 트윗 일부를 제공하며 다음과 같은 국가 데이터를 포함한다:

- Australia  
- Brazil  
- India  
- Indonesia  
- Japan  

각 CSV 파일에는 다음 4개의 컬럼이 포함된다.

| Column        | 설명 |
|---------------|------------------------------------|
| `id`          | 트윗 고유 ID |
| `created_at`  | 트윗 작성 시간 (문자열 형태) |
| `user_location` | 유저가 기재한 위치 정보 |
| `text`        | 트윗 내용 |

---

3. 데이터 처리 방식

### ✔ CSV 파일 로딩
프로젝트 실행 시 각 국가별 CSV 파일에서 다음 순서로 데이터를 읽는다.

1. 파일 존재 여부 확인  
2. 첫 줄을 제외한 모든 행을 반복 처리  
3. `split()`을 이용해 id, created_at, user_location, text를 분리  
4. created_at 문자열을 “YYYY-MM-DD” 형식 날짜로 변환  
5. Tweet 객체로 매핑하여 리스트에 저장  
6. 국가별 전체 트윗 수 계산  
7. 연-월 단위 그룹화 후 개수 계산  

4. 월별 트윗 수 계산

5. 분석 관찰
대부분의 국가에서 2021년 12월에 코로나 관련 관심이 크게 증가함
이는 연말 확진자 증가, 방역 정책 변화 등이 원인일 가능성이 있음
India는 2020년 10월에 독립적인 피크를 보임 (2차 감염 확산 시기와 연관 가능)
user_location은 문자열 기반이라 정제되지 않은 경우가 많았음
본 분석은 날짜 기반 트윗 분포만을 다룸(텍스트 감정 분석 등은 미포함)
